# Life Game Roguelike

ライフゲーム × ローグライクの Web ゲーム。プレイヤーはコンウェイのライフゲーム盤面に介入しながら、生き延びたターン数を競う。

## 起動

ES Modules を使っているため、`file://` ではなくローカルサーバ経由で開く必要がある。

```bash
# Python 3
python3 -m http.server 8000

# Node.js (npx)
npx serve

# それ以降は http://localhost:8000 をブラウザで開く
```

## ディレクトリ構成

```
life-game-roguelike/
├── index.html        # エントリーポイント

    ├── renderer.js   # Canvas2D 描画(差し替え可能)
    ├── audio.js      # Web Audio API 効果音
    ├── shop.js       # ショップアイテム定義
    └── storage.js    # ハイスコア永続化
```

## 操作方法

### マウス
- 左クリック: 配置 or 撃破
- 右クリック / Shift+クリック: 範囲撃破(3×3)
- ホバー: 配置プレビュー(緑=置ける、赤=置けない)

### キーボード
- `1`〜`6`: パターン切り替え(ブロック/ブリンカー/グライダー/LWSS/不死/繁殖)
- `Q`: 配置モード
- `E`: 撃破モード
- `R`: 90° 回転
- `F`: 左右反転
- `Space`: ターン進行

## ゲームルール

### 基本フロー

1. プレイヤーは初期所持金 40💰 と中央のブロック2つでスタート
2. パターン購入・配置 → ▶ターン進行 → 4ステップ進化 → 報酬・コスト精算 → 次ターン
3. お金がマイナスになった瞬間ゲームオーバー
4. スコア = ターン数 × 100 + 生存セル数 × 5

### セル種別

| セル | 色 | 説明 |
|------|----|----|
| 通常セル | 白 | 標準ライフゲームルール(隣接2-3で生存、3で誕生) |
| 不死セル | 緑 | 死なないが動かない。壁・アンカーとして使用 |
| 繁殖セル | 青 | 隣接2でも50%で誕生を促進。生存条件は2-4と緩い |
| 死亡ゾーン | 赤 | セルが入ると即死。動かない地雷 |
| 侵食セル | 紫 | 隣接した自セルを死亡させる。25%で拡散する敵 |

### 経済

- **報酬**: 生存セル数 × 1💰 を毎ターン獲得
- **維持コスト**: 毎ターン支払い。初期5💰、3ターンごとに+3💰
- **配置コスト**: ブロック5💰、ブリンカー4💰、グライダー8💰、LWSS18💰、不死15💰、繁殖10💰
- **撃破コスト**: 通常3💰、範囲撃破8💰

### 邪魔要素のスケジュール

- 偶数ターン: 死亡ゾーン1個追加
- 5ターンごと: 死亡ゾーン `2 + turn/5` 個追加
- 10ターン以降の5ターンごと: 侵食セル `1 + (turn-10)/10` 個追加

### ショップ

5ターンごとに行商人が登場、9種類のアイテムプールから3つランダム提示。1つ購入かスキップ。
報酬2倍・コスト軽減・ハザード一掃・浄化・無料グライダーなど。

## リスクとリターンの設計思想

### 3層構造

良いローグライクは大体これらの層が組み合わさっている。

**第1層: 持続的圧力(じわじわ系)**
何もしなければ確実に負ける、という基底の恐怖。
- 現状: 維持コスト上昇、ランダム邪魔配置
- 課題: ランダムなので「逃げ場が狭くなる」感覚が薄い

**第2層: 戦術的選択(毎ターン系)**
目先の最適解と長期の最適解が違う、という葛藤。
- 現状: 配置 vs 撃破 vs 貯金 の3択、ショップ選択
- 良好: ターンごとに考えることがある

**第3層: スパイク体験(瞬間系)**
「やったー!」「あぶなかった!」の瞬間。
- 現状: ほぼ無い。ショップで報酬2倍を引いた時くらい
- 課題: ここが薄いと「淡々と消化するゲーム」になる

### 強化したい体験

ユーザーフィードバックで明示された方向性:

1. **本能的な恐怖**: 敵が外側から迫ってくる、生存圏が狭まる
2. **ギリギリの一発逆転**: 追い詰められた状態から大破壊で逆転
3. **コンボの爆発的気持ち良さ**: 条件達成で大増殖、数の暴力

## 拡張アイデア(未実装)

### 🔥 優先度高: コアコンセプト補強

#### A. 外周侵食システム ⭐⭐⭐
**狙い**: 第1層を「ランダム不安」から「空間的恐怖」へ。

- ランダム邪魔配置を廃止
- 敵は必ず盤面の外周から発生
- ターン進行で「安全地帯」が中心に向かって縮む
- 安全地帯外には侵食セルとハザードがランダム配置

**実装ヒント**:
```js
// evolution.js or 新規 hazard.js
function getSafeZoneRadius(turn) {
  const maxRadius = Math.min(ROWS, COLS) / 2;
  const shrinkRate = 0.5; // ターンあたり
  return Math.max(3, maxRadius - turn * shrinkRate);
}
function isInSafeZone(r, c, turn) {
  const dr = r - ROWS / 2;
  const dc = c - COLS / 2;
  return Math.sqrt(dr*dr + dc*dc) < getSafeZoneRadius(turn);
}
// 描画時に安全地帯の境界線を半透明の円で表示
```

**注意点**: トロイダル盤面(端がつながる)との整合性を取る必要あり。
外周侵食なら盤面の端を「壁」扱いに変更したほうが直感的かもしれない。

#### B. ウェーブ予告システム ⭐⭐⭐
**狙い**: 緊張と弛緩のメリハリ。常時恐怖は麻痺するので「来るぞ」の瞬間を作る。

- 5ターンごとにウェーブ(侵食セル10個一気に出現)
- ウェーブ2ターン前に方向予告(「北から来ますわ!」など)
- 通常ターンは比較的平穏

**実装ヒント**: state に `nextWaveTurn`, `nextWaveDirection` を持たせる。
描画時に予告矢印を盤面端に表示。SFX で警告音も。

#### C. パターン認識ボーナス ⭐⭐⭐
**狙い**: 第3層、構築の喜び。ライフゲームの知識がゲーム的に報われる。

検出パターンと報酬の例:

| パターン | 検出方法 | 報酬 |
|---------|---------|------|
| グライダー誕生 | 5セルが正しい4向きどれかの形 | +20💰 + 効果音 |
| シューター完成 | グライダー砲のテンプレマッチ | +100💰 + ハザード一掃 |
| オシレーター | 同一形状が3周期以上維持 | 毎ターン報酬2倍 |
| プフ(無限増殖体) | 検出時に大ファンファーレ | +500💰 |

**実装ヒント**: 進化後の盤面をスキャンしてテンプレートマッチング。
各パターンは回転4×反転2=8パターン分のテンプレを用意。
重い場合は5×5窓のハッシュで高速化可能(セル状態をビットに詰めてMap参照)。

オシレーター検出は盤面ハッシュを過去N世代分保持して周期検出するのが王道。

#### D. チェインシステム ⭐⭐⭐
**狙い**: 第3層、爆発的増殖の気持ち良さ。

- 1ターン中(4ステップ進化中)の累計誕生数を計測
- 10誕生で×2倍、20誕生で×3倍...と倍率上昇
- ターン終了時の報酬に倍率適用
- チェイン中は画面端に大きな数字とエフェクト

**実装ヒント**: evolution.js の `evolve()` 戻り値 `births.length` を main.js で累積。
ターン跨ぎで0リセット。倍率は `1 + Math.floor(chainCount / 10)` など。

派手な演出のため、チェイン数に応じて画面シェイク・パーティクル・SFXピッチ上昇を仕込むと気持ち良い。

#### E. ラストスタンド ⭐⭐
**狙い**: 諦めない理由、逆転の可能性。

- 自セルが10個以下になると「絶望ボーナス」発動
- 報酬2倍、維持コスト半額
- 復活したら通常モードに戻る

**実装ヒント**: finishTurn() で生存セル数チェック、state.lastStandActive をトグル。
UIに「ラストスタンド!」表示と専用BGM切り替え演出。

### 🌟 優先度中: 補強アイデア

#### F. 借金システム
- お金がマイナスになっても1ターン猶予
- 次ターン終了時に支払えればOK
- 「あと1ターンで大爆発するから借金してでも維持」が成立
- マイナス時は画面が赤く点滅して緊張感

#### G. 必殺技ゲージ
- セル死亡で溜まる(破壊された数に応じて)
- MAX で「全画面範囲攻撃」「自セル全復活」「ハザード全消去」など
- 追い詰められるほど溜まる構造で逆転を仕込む
- ゲージUIは画面下に表示

#### H. リスクカード/契約
- ショップに「次5ターン維持コスト3倍、ただし報酬5倍」を追加
- 「次ターン侵食セル+5、ただし即座に+80💰」など能動的なハイリスクハイリターン

#### I. 自爆セル
- 配置するとNターン後に5×5範囲を全滅させる
- 自セルも巻き込むが、敵を一掃できる
- 追い詰められた時の捨て身の反撃
- カウントダウン数字をセル上に表示

#### J. 領土支配ボーナス
- 連続した自セル領域10個以上で「コロニー」報酬2倍
- 20個以上で「都市」報酬3倍
- 50個以上で「帝国」報酬5倍
- 大きく育てる楽しさ
- 実装: BFSで連結成分を計算

### 💡 優先度低: 装飾的アイデア

#### K. 共鳴セル(黄色セル)
- 盤面に4箇所配置すると囲まれた領域内のセル全部が報酬2倍

#### L. フィーバータイム
- 特定条件達成で10ターン「誕生条件が隣接2に緩和」
- 一気に増えるが侵食も加速

#### M. 統計グラフ
- 画面下にセル数・お金の推移グラフ
- プレイ振り返り要素

#### N. メタ進化(プレイ間アンロック)
- ゲームオーバー時のスコアでポイント獲得
- 「初期所持金+10」「グライダーコスト-1」など永続強化
- 「新パターン解放」「新敵セル解禁」でゲーム性自体が変わる

### 🎯 推奨実装プラン

**Phase 1: 恐怖の核を作る(A + B)**
外周侵食とウェーブ予告で「迫り来る敵」の体験を確立。
これが入ると現状のランダム配置とは別ゲームになる。

**Phase 2: 楽しさの核を作る(C + D)**
パターン認識とチェインで「育てれば化ける」を実装。
Phase 1 とセットになって「追い詰められる→必死で耐える→コンボ発火で大逆転」のドラマが生まれる。

**Phase 3: 逆転要素を仕込む(E + G)**
ラストスタンドと必殺技ゲージで「諦めない理由」を作る。

**Phase 4以降: バランス調整しつつF/H/I/Jから選んで追加**

### 設計上の注意点

#### 「外側から迫る恐怖」と「コンボ増殖」の両立
外周侵食で安全地帯が狭まる + コンボで爆発的増殖が起きると、
増殖したセルがすぐ侵食に飲まれてもったいない展開になる可能性がある。

**緩和案**:
- コンボ発動中は侵食速度が一時的に低下
- ラストスタンド中は安全地帯が一時的に広がる
- 大きく育てた領土は侵食に対して耐性を持つ(連結数が多いほど死ににくい)

#### 「ギリギリ感」を本物にするために
追い詰められた状態を演出だけでなく実際に発生させる必要がある。

**緩和案**:
- 序盤の難易度を上げて「楽勝ターン」を作らない
- 中盤に強力な選択肢を与えて「攻めるか守るか」の判断を迫る
- 終盤は維持コストが指数的に上昇して長期生存が物理的に困難に

## PixiJS への移行

`js/renderer.js` の `CanvasRenderer` と同じインターフェース(`draw`, `markBirth`, `markDeath`, `setHover`, `reset`, `mouseToGrid`)を持つ `PixiRenderer` を実装し、`js/main.js` の import 行を差し替えるだけで移行できる。

```js
// 例: js/main.js
// import { CanvasRenderer } from './renderer.js';
import { PixiRenderer as CanvasRenderer } from './pixi-renderer.js';
```

PixiJS なら 1200セルそれぞれを `PIXI.Sprite` として保持し、変化したセルだけ `alpha` と `scale` を更新する差分レンダリングで高速化できる。
チェインのエフェクト、パターン認識時のフラッシュ、パーティクル演出など派手な見た目も簡単。

### セットアップ手順

```bash
npm init -y
npm i pixi.js
npm i -D vite
# vite.config.js を作成、index.html を src/ に移動
npx vite
```

## 既知の調整ポイント

### バランス
- 撃破コスト 3💰: 安すぎる可能性あり。整地が容易すぎるとパターン配置の戦略性が薄まる
- ブロックの優位性: 4セルで安定報酬源になるため、他のパターンより使われがち
- リアルタイム介入の速度感: デフォルト 240ms はやや速い体感かも
- 維持コストの上昇カーブ: 3ターンごとに+3の線形だが、後半は指数的にした方が緊張感が出るかも

### 拡張時に注意
- 新しいセル種別を追加する時は constants.js + evolution.js + renderer.js(色マップ)の3箇所更新
- ショップアイテムは shop.js の SHOP_POOL に追記するだけ
- パターン認識を実装する時は evolution.js を変更せず、main.js で進化後盤面をスキャンする形が綺麗
- 大規模変更(外周侵食など)時は hazard.js のような新モジュールに切り出すのが望ましい
